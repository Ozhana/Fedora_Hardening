| **Purpose and Philosophy** <br><br> Standard Linux users rely on basic aliases to save time (e.g., `alias ll='ls -al'`). This suite is **not** for standard users. This repository provides an "Enterprise-Grade" architectural layer embedded directly into your `.bashrc` and `/usr/local/bin`. It operates on the **"Don't Assume, Verify"** and **"Zero-Trust"** engineering principles. <br><br> The goal is to give system administrators, security engineers, and forensic analysts atomic control over kernel modules, deterministic telemetry (via AWK), and human‑in‑the‑loop baseline validation.<br><br> **How It Hardens Your System (Defense in Depth)**<br><br>* **Kernel Isolation:** Disables USB storage at the `modprobe` level. Even if a physical device bypasses hardware filters, the OS cannot mount or read the file system.<br><br>* **Immutable Architecture:** Enforces Ext4/Btrfs `chattr +i` locks on critical files. Even compromised `root` accounts cannot alter or delete your sealed configurations.<br><br>* **Baseline Poisoning Prevention:** Implements a strict "Human-in-the-loop" approval flow for Rkhunter and AIDE, preventing automated systems from accidentally signing malicious binaries as safe.<br><br>* **Forensic-Grade Destruction:** Destroys sensitive files using the 3-pass Gutmann/DoD method with a final zero-fill (`shred -u -z -n 3`).<br><br>* **Idempotent & Atomic Scripts:** Every script uses `flock`‑based locking, `trap` cleanup, and strict `bash` mode (`set -Eeuo pipefail`). They can be executed 1000 times without breaking the system or flooding logs. | **Amaç ve Felsefe** <br><br> Standart Linux kullanıcıları zaman kazanmak için basit kısayollar kullanır. ## 🧬 Technical Hardening Matrix / Teknik Sıkılaştırma Matrisi

| Component | Attack Mitigated | Implementation |
|-----------|-----------------|----------------|
| `secure-sysupdate` | Man‑in‑the‑middle package injection | `flock`‑locked `dnf upgrade` with `--refresh` |
| `secure-cleancache` | Log‑based evidence tampering | Atomic `journalctl --vacuum-time=7d` |
| `secure-usbctl` | USB‑borne malware / BadUSB | Kernel module removal (`modprobe -r usb-storage uas`) |
| `sys-chattr` | Ransomware file encryption | Immutable flag (`chattr +i`) on critical assets |
| `secure-rkhunter` | Rootkit persistence | Human‑approved `--propupd` to prevent baseline poisoning |
| `secure-aide` | Stealthy binary replacement | File integrity checksum database with manual update |
| `secure-wipe` | Forensic data recovery | 3‑pass shred with final zero‑fill (`shred -u -z -n 3`) |
| `sys-ramradar` / `memstat` | Resource exhaustion DoS | Deterministic AWK‑based telemetry |
| `sys-thermal` | Thermal throttling / hardware damage | On‑demand `sensors` readout |
| `secure-needsrestart` | Vulnerable legacy libraries in running services | `dnf needs-restarting -r` post‑update scan |
| `sys-suidscan` | Privilege escalation via SUID/SGID | Curated scan of `/usr`, `/bin`, `/sbin`, `/lib*`, `/opt` |
| `secure-permcheck` | Unauthorised access to SSH keys & config | Checks `$HOME` (700), `~/.ssh` (700), `~/.bashrc` (644) |
| `trap cleanup` in every script | Zombie lock files after `CTRL+C` / signal | Signal‑safe `flock` release + PID file removal |
| `flock`‑based atomic locks | Race condition / duplicate execution | Kernel‑level exclusive file lock with PID stamp |
| `set -Eeuo pipefail` / `IFS=$'\n\t'` | Silent failures / argument splitting bugs | Strict Bash execution environment |
