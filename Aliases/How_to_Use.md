| **Purpose and Philosophy**<br><br>This suite is **not** for average users. It embeds an enterprise‑grade security layer directly into your Bash environment, following the principles of **"Don't Assume, Verify"** and **Zero‑Trust**. The goal is to give system administrators, security engineers, and forensic analysts atomic control over kernel modules, deterministic telemetry, and human‑in‑the‑loop baseline validation.<br><br>**Key Hardening Features:**<br>• **Kernel Isolation** – USB storage drivers can be unloaded to prevent data exfiltration.<br>• **Immutable Architecture** – Files can be locked with `chattr +i` so even root cannot modify them.<br>• **Baseline Poisoning Prevention** – Rkhunter and AIDE require explicit human approval before updating their trusted databases.<br>• **Forensic‑Grade Destruction** – Sensitive files are shredded with 3 passes and a final zero‑fill (`shred -u -z -n 3`).<br>• **Idempotent & Atomic Scripts** – Every function uses `flock` locks, `trap` cleanup, and strict Bash mode (`set -Eeuo pipefail`). They can run 1000 times without side effects. | **Amaç ve Felsefe**<br><br>Bu paket sıradan kullanıcılar için **değildir**. **“Varsayımda Bulunma, Doğrula”** ve **Sıfır Güven** prensipleriyle çalışan kurumsal düzey bir güvenlik katmanını Bash ortamınıza yerleştirir. ## 🔄 Recovery & Rollback / Kurtarma ve Geri Alma

| 🇬🇧 ENGLISH | 🇹🇷 TÜRKÇE |
|------|------|
| To completely revert all changes:<br><br>1. Restore original `.bashrc`:<br><code>cp ~/.bashrc.bak.TIMESTAMP ~/.bashrc</code><br>2. Remove deployed scripts:<br><code>sudo rm -f /usr/local/bin/secure-* /usr/local/bin/sys-*</code><br>3. İsteğe bağlı olarak kilit dosyalarını ve logları temizleyin:<br><code>rm -rf ~/.local/run/locks/secure* ~/Desktop/LOG_FILES/</code><br><br>**Not:** `/usr/local/bin` dışında hiçbir sistem dosyası değiştirilmez. ## 🧬 Technical Hardening Matrix / Teknik Sıkılaştırma Matrisi

| Component | Attack Mitigated | Implementation |
|-----------|-----------------|----------------|
| `secure-sysupdate` | Man-in-the-middle package injection | `flock`‑locked `dnf upgrade --refresh` |
| `secure-cleancache` | Log tampering / disk exhaustion | Atomic `journalctl --vacuum-time=7d` |
| `secure-usbctl` | USB‑borne malware / BadUSB | Kernel module removal (`modprobe -r usb-storage uas`) |
| `sys-chattr` | Ransomware file encryption | Immutable flag (`chattr +i`) on critical assets |
| `secure-rkhunter` | Rootkit persistence | Human‑approved `--propupd` (baseline poisoning prevention) |
| `secure-aide` | Stealthy binary replacement | File integrity checksum database with manual update |
| `secure-wipe` | Forensic data recovery | 3‑pass shred with final zero‑fill (`shred -u -z -n 3`) |
| `sys-ramradar` / `memstat` | Resource exhaustion DoS | Deterministic AWK‑based telemetry |
| `sys-thermal` | Thermal throttling / hardware damage | On‑demand `sensors` readout |
| `secure-needsrestart` | Vulnerable legacy libraries in running services | `dnf needs-restarting -r` post‑update scan |
| `sys-suidscan` | Privilege escalation via SUID/SGID | Curated scan of `/usr`, `/bin`, `/sbin`, `/lib*`, `/opt` |
| `secure-permcheck` | Unauthorised access to SSH keys & config | Checks `$HOME` (700), `~/.ssh` (700), `~/.bashrc` (644) |
| `trap cleanup` in every script | Zombie lock files after `CTRL+C` / signal | Signal‑safe `flock` release + PID file removal |
| `flock`‑based atomic locks | Race condition / duplicate execution | Kernel‑level exclusive file lock with PID stamp |
| `set -Eeuo pipefail` / `IFS=$'\n\t'` | Silent failures / argument splitting bugs | Strict Bash execution environment |
