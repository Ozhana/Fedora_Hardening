# Aegis AIDE Shield
## Zero-Trust Asynchronous Intrusion Detection Architecture (V18.1)

| ENGLISH | TÜRKÇE |
| :--- | :--- |
| **Introduction**<br>This project integrates AIDE (Advanced Intrusion Detection Environment) into Fedora Linux using a Red Team zero-trust approach. Optimized for V18.1, it decouples heavy database updates from DNF transactions to save NVMe SSD life, utilizing state-aware Systemd timers, atomic BTRFS syncs, and strict sandboxing. | **Giriş**<br>Bu proje, AIDE'yi (Gelişmiş Saldırı Tespit Ortamı) Kırmızı Takım sıfır-güven yaklaşımıyla Fedora Linux'a entegre eder. V18.1 için optimize edilen bu yapı, ağır veritabanı güncellemelerini DNF işlemlerinden ayırarak NVMe SSD ömrünü korur; durum-farkındalıklı Systemd zamanlayıcıları, atomik BTRFS senkronizasyonları ve sıkı kum havuzu (sandboxing) önlemleri kullanır. |
| **Step 1: Install Dependencies**<br>First, install AIDE and the required DNF transaction hook plugin to track package shifts.<br><br><code>sudo dnf install -y aide python3-dnf-plugin-post-transaction-actions</code><br><br>Initialize the baseline database:<br><code>sudo aide --init</code><br><code>sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz</code> | **Adım 1: Bağımlılıkları Kurun**<br>Öncelikle AIDE'yi ve paket değişikliklerini izlemek için gerekli DNF işlem kancası (transaction hook) eklentisini kurun.<br><br><code>sudo dnf install -y aide python3-dnf-plugin-post-transaction-actions</code><br><br>İlk temel çizgi (baseline) veritabanını oluşturun:<br><code>sudo aide --init</code><br><code>sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz</code> |
| **Step 2: Prepare Workspace**<br>Create an isolated space to pull your hardened scripts from GitHub before installing them to privileged paths.<br><br><code>mkdir -p ~/aegis-aide-installer && cd ~/aegis-aide-installer</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/AIDE/aegis-aide.sh</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/AIDE/aegis-aide.service</code> | **Adım 2: Çalışma Alanını Hazırlayın**<br>Betikleri yetkili sistem yollarına yüklemeden önce GitHub'dan çekmek için izole bir alan oluşturun.<br><br><code>mkdir -p ~/aegis-aide-installer && cd ~/aegis-aide-installer</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/AIDE/aegis-aide.sh</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/AIDE/aegis-aide.service</code> |
| **Step 3: The DNF Flag Hook**<br>Create the lightweight script that flags a DNF modification instantly without freezing the current transaction.<br><br><code>sudo nano /usr/local/bin/aegis-aide-flag.sh</code><br><br>Add content:<br><code>#!/usr/bin/env bash</code><br><code>touch /run/aegis_aide_pending.flag</code><br><code>exit 0</code><br><br>Secure it and link to DNF:<br><code>sudo chmod 700 /usr/local/bin/aegis-aide-flag.sh</code><br><code>sudo mkdir -p /etc/dnf/plugins/post-transaction-actions.d/</code><br><code>echo "*:in,out,up,rm:/usr/local/bin/aegis-aide-flag.sh" \| sudo tee /etc/dnf/plugins/post-transaction-actions.d/aide.action</code> | **Adım 3: DNF Bayrak Kancası**<br>Mevcut işlemi dondurmadan bir DNF değişikliğini anında işaretleyen hafif betiği oluşturun.<br><br><code>sudo nano /usr/local/bin/aegis-aide-flag.sh</code><br><br>İçeriği ekleyin:<br><code>#!/usr/bin/env bash</code><br><code>touch /run/aegis_aide_pending.flag</code><br><code>exit 0</code><br><br>Güvenliğe alın ve DNF'e bağlayın:<br><code>sudo chmod 700 /usr/local/bin/aegis-aide-flag.sh</code><br><code>sudo mkdir -p /etc/dnf/plugins/post-transaction-actions.d/</code><br><code>echo "*:in,out,up,rm:/usr/local/bin/aegis-aide-flag.sh" \| sudo tee /etc/dnf/plugins/post-transaction-actions.d/aide.action</code> |
| **Step 4: The Main Aegis Script**<br>Move your production `aegis-aide.sh` to the designated bin directory and lock file access permissions.<br><br><code>sudo cp aegis-aide.sh /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chown root:root /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chmod 700 /usr/local/bin/aegis-aide-update.sh</code> | **Adım 4: Ana Aegis Betiği**<br>Üretim ortamı için hazırlanan `aegis-aide.sh` dosyasını belirlenen bin dizinine taşıyın ve dosya erişim izinlerini kilitleyin.<br><br><code>sudo cp aegis-aide.sh /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chown root:root /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chmod 700 /usr/local/bin/aegis-aide-update.sh</code> |
| **Step 5: Systemd Configuration**<br>Deploy the primary hardened service and establish the asynchronous weekly execution timer.<br><br><code>sudo cp aegis-aide.service /etc/systemd/system/aegis-aide.service</code><br><code>sudo nano /etc/systemd/system/aegis-aide.timer</code><br><br>Add timer rules:<br><code>[Unit]</code><br><code>Description=AEGIS AIDE Weekly SSD-Friendly Timer</code><br><code>[Timer]</code><br><code>OnCalendar=Sun *-*-* 00:00:00</code><br><code>Persistent=true</code><br><code>[Install]</code><br><code>WantedBy=timers.target</code><br><br>Activate:<br><code>sudo systemctl daemon-reload</code><br><code>sudo systemctl enable --now aegis-aide.timer</code> | **Adım 5: Systemd Yapılandırması**<br>Katılaştırılmış ana servisi dağıtın ve asenkron haftalık çalışma zamanlayıcısını oluşturun.<br><br><code>sudo cp aegis-aide.service /etc/systemd/system/aegis-aide.service</code><br><code>sudo nano /etc/systemd/system/aegis-aide.timer</code><br><br>Zamanlayıcı kurallarını ekleyin:<br><code>[Unit]</code><br><code>Description=AEGIS AIDE Weekly SSD-Friendly Timer</code><br><code>[Timer]</code><br><code>OnCalendar=Sun *-*-* 00:00:00</code><br><code>Persistent=true</code><br><code>[Install]</code><br><code>WantedBy=timers.target</code><br><br>Aktifleştirin:<br><code>sudo systemctl daemon-reload</code><br><code>sudo systemctl enable --now aegis-aide.timer</code> |
| **Step 6: Initialize Audit Log & Security**<br>Create the dedicated log audit file, restrict reading capabilities, and fix SELinux labeling to prevent permission blocks.<br><br><code>sudo touch /var/log/aegis_aide_audit.log</code><br><code>sudo chmod 0600 /var/log/aegis_aide_audit.log</code><br><code>sudo restorecon -v /var/log/aegis_aide_audit.log</code> | **Adım 6: Denetim Logunu ve Güvenliği Başlatın**<br>Özel log denetim dosyasını oluşturun, okuma yetkilerini kısıtlayın ve izin engellemelerini önlemek için SELinux etiketlerini düzeltin.<br><br><code>sudo touch /var/log/aegis_aide_audit.log</code><br><code>sudo chmod 0600 /var/log/aegis_aide_audit.log</code><br><code>sudo restorecon -v /var/log/aegis_aide_audit.log</code> |
| **Step 7: Logrotate Setup**<br>Prevent log inflation from exhausting storage capacity by establishing strict rotation schedules.<br><br><code>sudo nano /etc/logrotate.d/aegis-aide</code><br><br>Content:<br><code>/var/log/aegis_aide_audit.log {</code><br><code>  weekly</code><br><code>  rotate 4</code><br><code>  compress</code><br><code>  delaycompress</code><br><code>  missingok</code><br><code>  notifempty</code><br><code>  create 0600 root root</code><br><code>}</code> | **Adım 7: Logrotate Kurulumu**<br>Sıkı rotasyon takvimleri belirleyerek log şişmesinin depolama kapasitesini tüketmesini önleyin.<br><br><code>sudo nano /etc/logrotate.d/aegis-aide</code><br><br>İçerik:<br><code>/var/log/aegis_aide_audit.log {</code><br><code>  weekly</code><br><code>  rotate 4</code><br><code>  compress</code><br><code>  delaycompress</code><br><code>  missingok</code><br><code>  notifempty</code><br><code>  create 0600 root root</code><br><code>}</code> |
| **⚠️ CRITICAL: SYSTEMD HARDENING NOTICE**<br>The `aegis-aide.service` file includes advanced security controls like `MemoryDenyWriteExecute=yes` and `PrivateDevices=yes`. While these strictly follow Red Team deployment standards, compatibility may vary depending on your specific AIDE version, compilation flags, or SELinux policies.<br><br>**Verification:** If AIDE fails to execute, manually test it:<br><code>sudo touch /run/aegis_aide_pending.flag</code><br><code>sudo systemctl start aegis-aide.service</code><br><code>sudo journalctl -u aegis-aide.service -b --no-pager</code><br><br>If you see a memory allocation restriction or NAMESPACE error, change `MemoryDenyWriteExecute=no` in your service file and run `sudo systemctl daemon-reload`. | **⚠️ KRİTİK: SYSTEMD KANUNU VE UYARISI**<br>`aegis-aide.service` dosyası, `MemoryDenyWriteExecute=yes` ve `PrivateDevices=yes` gibi gelişmiş güvenlik kontrolleri içerir. Bunlar Kırmızı Takım dağıtım standartlarına kesinlikle uysa da; özel AIDE sürümünüze, derleme bayraklarına veya SELinux politikalarına bağlı olarak uyumluluk değişebilir.<br><br>**Doğrulama:** AIDE yürütme hatası verirse manuel test edin:<br><code>sudo touch /run/aegis_aide_pending.flag</code><br><code>sudo systemctl start aegis-aide.service</code><br><code>sudo journalctl -u aegis-aide.service -b --no-pager</code><br><br>Eğer bir bellek kısıtlama veya NAMESPACE hatası görürseniz, servis dosyanızdaki `MemoryDenyWriteExecute=no` ayarını değiştirip `sudo systemctl daemon-reload` komutunu çalıştırın. |
