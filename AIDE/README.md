# Aegis AIDE Shield
## Zero-Trust Asynchronous Intrusion Detection Architecture

| ENGLISH | TÜRKÇE |
| :--- | :--- |
| **Introduction**<br>This project integrates AIDE (Advanced Intrusion Detection Environment) into Fedora Linux using a Red Team zero-trust approach. It decouples heavy database updates from DNF transactions to save NVMe SSD life, utilizing state-aware Systemd timers, atomic BTRFS syncs, and strict hardening. | **Giriş**<br>Bu proje, AIDE'yi (Gelişmiş Saldırı Tespit Ortamı) Kırmızı Takım sıfır-güven yaklaşımıyla Fedora Linux'a entegre eder. Ağır veritabanı güncellemelerini DNF işlemlerinden ayırarak NVMe SSD ömrünü korur; durum-farkındalıklı Systemd zamanlayıcıları, atomik BTRFS senkronizasyonları ve sıkı güvenlik katılaştırmaları kullanır. |
| **Step 1: Install Dependencies**<br>First, install AIDE and the required DNF plugin.<br><br><code>sudo dnf install -y aide python3-dnf-plugin-post-transaction-actions</code><br><br>Initialize the baseline if you haven't already:<br><code>sudo aide --init</code><br><code>sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz</code> | **Adım 1: Bağımlılıkları Kurun**<br>Öncelikle AIDE ve gerekli DNF eklentisini kurun.<br><br><code>sudo dnf install -y aide python3-dnf-plugin-post-transaction-actions</code><br><br>Eğer henüz yapmadıysanız ilk temel çizgiyi (baseline) oluşturun:<br><code>sudo aide --init</code><br><code>sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz</code> |
| **Step 2: Prepare Workspace**<br>Create an isolated folder in your home directory to download and place the files before moving them to system directories.<br><br><code>cd ~</code><br><code>mkdir aegis-aide-installer</code><br><code>cd aegis-aide-installer</code> | **Adım 2: Çalışma Alanını Hazırlayın**<br>Dosyaları sistem dizinlerine taşımadan önce indirmek ve düzenlemek için ev dizininizde (home) izole bir klasör oluşturun.<br><br><code>cd ~</code><br><code>mkdir aegis-aide-installer</code><br><code>cd aegis-aide-installer</code> |
| **Step 3: The DNF Flag Hook**<br>Create the lightweight script that signals a DNF transaction occurred without hanging the system.<br><br><code>sudo nano /usr/local/bin/aegis-aide-flag.sh</code><br><br>Add the following content:<br><code>#!/usr/bin/env bash</code><br><code>touch /run/aegis_aide_pending.flag</code><br><code>exit 0</code><br><br>Make it executable:<br><code>sudo chmod 700 /usr/local/bin/aegis-aide-flag.sh</code><br><br>Register the hook with DNF:<br><code>sudo mkdir -p /etc/dnf/plugins/post-transaction-actions.d/</code><br><code>echo "*:in,out,up,rm:/usr/local/bin/aegis-aide-flag.sh" \| sudo tee /etc/dnf/plugins/post-transaction-actions.d/aide.action</code> | **Adım 3: DNF Bayrak Kancası**<br>Sistemi kilitlemeden DNF işleminin gerçekleştiğini işaretleyen hafif betiği oluşturun.<br><br><code>sudo nano /usr/local/bin/aegis-aide-flag.sh</code><br><br>Şu içeriği ekleyin:<br><code>#!/usr/bin/env bash</code><br><code>touch /run/aegis_aide_pending.flag</code><br><code>exit 0</code><br><br>Çalıştırılabilir yapın:<br><code>sudo chmod 700 /usr/local/bin/aegis-aide-flag.sh</code><br><br>Kancayı DNF'e kaydedin:<br><code>sudo mkdir -p /etc/dnf/plugins/post-transaction-actions.d/</code><br><code>echo "*:in,out,up,rm:/usr/local/bin/aegis-aide-flag.sh" \| sudo tee /etc/dnf/plugins/post-transaction-actions.d/aide.action</code> |
| **Step 4: The Main Aegis Script**<br>Move your saved <code>aegis-aide.sh</code> to the correct location and secure it:<br><br><code>sudo cp aegis-aide.sh /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chown root:root /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chmod 700 /usr/local/bin/aegis-aide-update.sh</code> | **Adım 4: Ana Aegis Betiği**<br>Kaydettiğiniz <code>aegis-aide.sh</code> dosyasını doğru konuma taşıyın ve güvenliğini sağlayın:<br><br><code>sudo cp aegis-aide.sh /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chown root:root /usr/local/bin/aegis-aide-update.sh</code><br><code>sudo chmod 700 /usr/local/bin/aegis-aide-update.sh</code> |
| **Step 5: Systemd Configuration**<br>Move your saved <code>aegis-aide.service</code> file to the systemd directory:<br><br><code>sudo cp aegis-aide.service /etc/systemd/system/aegis-aide.service</code><br><br>Create the weekly timer:<br><code>sudo nano /etc/systemd/system/aegis-aide.timer</code><br><br>Add this content:<br><code>[Unit]</code><br><code>Description=AEGIS AIDE Weekly SSD-Friendly Timer</code><br><code>[Timer]</code><br><code>OnCalendar=Sun *-*-* 00:00:00</code><br><code>Persistent=true</code><br><code>[Install]</code><br><code>WantedBy=timers.target</code><br><br>Enable the timer:<br><code>sudo systemctl daemon-reload</code><br><code>sudo systemctl enable --now aegis-aide.timer</code> | **Adım 5: Systemd Yapılandırması**<br>Kaydettiğiniz <code>aegis-aide.service</code> dosyasını systemd dizinine taşıyın:<br><br><code>sudo cp aegis-aide.service /etc/systemd/system/aegis-aide.service</code><br><br>Haftalık zamanlayıcıyı oluşturun:<br><code>sudo nano /etc/systemd/system/aegis-aide.timer</code><br><br>Şu içeriği ekleyin:<br><code>[Unit]</code><br><code>Description=AEGIS AIDE Weekly SSD-Friendly Timer</code><br><code>[Timer]</code><br><code>OnCalendar=Sun *-*-* 00:00:00</code><br><code>Persistent=true</code><br><code>[Install]</code><br><code>WantedBy=timers.target</code><br><br>Zamanlayıcıyı etkinleştirin:<br><code>sudo systemctl daemon-reload</code><br><code>sudo systemctl enable --now aegis-aide.timer</code> |
| **Step 6: Initialize Audit Log**<br>Create the audit log file securely and set proper SELinux contexts so the service doesn't fail:<br><br><code>sudo touch /var/log/aegis_aide_audit.log</code><br><code>sudo chmod 0600 /var/log/aegis_aide_audit.log</code><br><code>sudo restorecon -v /var/log/aegis_aide_audit.log</code> | **Adım 6: Denetim Logunu (Audit Log) Başlatın**<br>Denetim log dosyasını güvenli bir şekilde oluşturun ve servisin çökmemesi için uygun SELinux etiketlerini ayarlayın:<br><br><code>sudo touch /var/log/aegis_aide_audit.log</code><br><code>sudo chmod 0600 /var/log/aegis_aide_audit.log</code><br><code>sudo restorecon -v /var/log/aegis_aide_audit.log</code> |
| **Step 7: Logrotate Setup**<br>Prevent your SSD from filling up with old logs by adding a logrotate rule:<br><br><code>sudo nano /etc/logrotate.d/aegis-aide</code><br><br>Content:<br><code>/var/log/aegis_aide_audit.log {</code><br><code>  weekly</code><br><code>  rotate 4</code><br><code>  compress</code><br><code>  delaycompress</code><br><code>  missingok</code><br><code>  notifempty</code><br><code>  create 0600 root root</code><br><code>}</code> | **Adım 7: Logrotate Kurulumu**<br>Eski logların SSD'nizi doldurmasını önlemek için bir logrotate kuralı ekleyin:<br><br><code>sudo nano /etc/logrotate.d/aegis-aide</code><br><br>İçerik:<br><code>/var/log/aegis_aide_audit.log {</code><br><code>  weekly</code><br><code>  rotate 4</code><br><code>  compress</code><br><code>  delaycompress</code><br><code>  missingok</code><br><code>  notifempty</code><br><code>  create 0600 root root</code><br><code>}</code> |
| **⚠️ CRITICAL: SYSTEMD HARDENING NOTICE**<br>The <code>aegis-aide.service</code> file includes advanced security controls like <code>MemoryDenyWriteExecute=yes</code> and <code>PrivateDevices=yes</code>. While these strictly follow Red Team deployment standards, compatibility may vary depending on your specific AIDE version, compilation flags, or SELinux policies.<br><br>**Verification:** If AIDE fails to execute, manually test it:<br><code>sudo touch /run/aegis_aide_pending.flag</code><br><code>sudo systemctl start aegis-aide.service</code><br><code>sudo journalctl -u aegis-aide.service -b --no-pager</code><br><br>If you see a memory allocation restriction or NAMESPACE error, change <code>MemoryDenyWriteExecute=no</code> and run <code>sudo systemctl daemon-reload</code>. | **⚠️ KRİTİK: SYSTEMD KANUNU VE UYARISI**<br><code>aegis-aide.service</code> dosyası, <code>MemoryDenyWriteExecute=yes</code> ve <code>PrivateDevices=yes</code> gibi gelişmiş güvenlik kontrolleri içerir. Bunlar Kırmızı Takım dağıtım standartlarına kesinlikle uysa da; özel AIDE sürümünüze, derleme bayraklarına veya SELinux politikalarına bağlı olarak uyumluluk değişebilir.<br><br>**Doğrulama:** AIDE yürütme hatası verirse manuel test edin:<br><code>sudo touch /run/aegis_aide_pending.flag</code><br><code>sudo systemctl start aegis-aide.service</code><br><code>sudo journalctl -u aegis-aide.service -b --no-pager</code><br><br>Eğer bir bellek kısıtlama veya NAMESPACE hatası görürseniz, <code>MemoryDenyWriteExecute=no</code> ayarını değiştirip <code>sudo systemctl daemon-reload</code> komutunu çalıştırın. |
