# Aegis USB Storage Isolation Protocol

Zero-Trust USB Device Storage Authorization Framework

| ENGLISH | TÜRKÇE |
| :--- | :--- |
| **Enterprise-Grade USB Storage Defense**<br>This repository contains the V10.2 LTS protocol for hardware-level USB storage isolation. It separates the hardware firewall (USBGuard) from the logical driver layer. | **Kurumsal Düzey USB Depolama Savunması**<br>Bu depo, donanım seviyesinde USB depolama izolasyonu için V10.2 LTS protokolünü içerir. Donanım güvenlik duvarını (USBGuard) mantıksal sürücü katmanından ayırır. |
| **The Philosophy**<br>USBGuard blocks unauthorized devices at the port level, but if a device is authorized, its storage drivers load automatically. This protocol blacklists `usb-storage` and `uas` modules by default, requiring explicit, intentional activation (`usb-ac`) and allowing surgical decoupling (`usb-kapat`) to reduce the attack surface. | **Felsefe**<br>USBGuard yetkisiz cihazları engeller, ancak yetkili bir cihazda depolama sürücüleri otomatik yüklenir. Bu protokol, `usb-storage` ve `uas` modüllerini varsayılan olarak kara listeye alır, bilinçli aktivasyon (`usb-ac`) gerektirir ve saldırı yüzeyini daraltmak için cerrahi söküm (`usb-kapat`) sağlar. |
| **Features**<br>- Zero-Trust Architecture<br>- Kernel Module Blacklisting<br>- Systemd Audit Logging<br>- Deterministic Unmount & Sync | **Özellikler**<br>- Sıfır Güven (Zero-Trust) Mimarisi<br>- Çekirdek Modülü Kara Listesi<br>- Systemd Denetim Günlüklemesi (Audit Logging)<br>- Deterministik Söküm ve Senkronizasyon |
| **Step 1: Blacklist Kernel Modules**<br>Prevent auto-loading of storage drivers. Run this once in your terminal to seal the kernel:<br>`echo -e "blacklist usb-storage\nblacklist uas" \| sudo tee /etc/modprobe.d/usb-storage-aegis.conf` | **Adım 1: Çekirdek Modüllerini Kara Listeye Alın**<br>Depolama sürücülerinin otomatik yüklenmesini engelleyin. Çekirdeği mühürlemek için bir kez çalıştırın:<br>`echo -e "blacklist usb-storage\nblacklist uas" \| sudo tee /etc/modprobe.d/usb-storage-aegis.conf` |
| **Step 2: Installation**<br>Download the scripts (`usb-kapat` and `usb-ac`) and place them in the secure binary path:<br>`sudo cp usb-kapat /usr/local/bin/`<br>`sudo cp usb-ac /usr/local/bin/`<br>`sudo chmod 700 /usr/local/bin/usb-kapat /usr/local/bin/usb-ac` | **Adım 2: Kurulum**<br>Betikleri (`usb-kapat` ve `usb-ac`) indirin ve güvenli çalıştırma dizinine taşıyın:<br>`sudo cp usb-kapat /usr/local/bin/`<br>`sudo cp usb-ac /usr/local/bin/`<br>`sudo chmod 700 /usr/local/bin/usb-kapat /usr/local/bin/usb-ac` |
| **Step 3: Usage (usb-kapat)**<br>Flushes RAM cache to disk, unmounts USB drives deterministically without breaking root structures, and removes storage modules from the kernel. Run:<br>`sudo usb-kapat` | **Adım 3: Kullanım (usb-kapat)**<br>RAM önbelleğini diske yazar, kök dizini bozmadan USB diskleri kararlı şekilde unmount eder ve depolama modüllerini çekirdekten kazır. Çalıştırın:<br>`sudo usb-kapat` |
| **Step 4: Usage (usb-ac)**<br>Bypasses the blacklist to manually load storage drivers only when you intentionally need to read a USB drive. Run:<br>`sudo usb-ac` | **Adım 4: Kullanım (usb-ac)**<br>Sadece bilinçli olarak bir USB okumanız gerektiğinde, kara listeyi ezip geçerek depolama sürücülerini yükler. Çalıştırın:<br>`sudo usb-ac` |
| **Step 5: Audit & Telemetry**<br>Every action (module load/unload state) is securely logged to the systemd journal. To monitor USB storage events and prove zero-entropy, run:<br>`journalctl -t aegis-usb` | **Adım 5: Denetim ve Telemetri**<br>Her işlem (modül yükleme/sökme durumu) systemd günlüğüne güvenle kaydedilir. USB depolama olaylarını izlemek ve sıfır-entropiyi doğrulamak için çalıştırın:<br>`journalctl -t aegis-usb` |
