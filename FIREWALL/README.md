# Aegis Firewall Lockdown
## Enterprise-Grade Docker-Optimized Network Hardening (V7.2)

| ENGLISH | TÜRKÇE |
| :--- | :--- |
| **Introduction**<br>This repository houses **Aegis Firewall Lockdown V7.2**, a production-hardened network shield specifically optimized for Fedora 44 and Surface Pro 9 environments. It enforces strict zero-trust egress/ingress filtering, stateful inspection, and seamlessly coexists with Docker's dynamic iptables/nftables manipulation without breaking container networking. | **Giriş**<br>Bu depo, Fedora 44 ve Surface Pro 9 ortamları için özel olarak optimize edilmiş, üretim ortamına hazır bir ağ kalkanı olan **Aegis Firewall Lockdown V7.2**'yi barındırır. Sıkı sıfır-güven çıkış/giriş (egress/ingress) filtrelemesi, durum-farkındalıklı denetim uygular ve konteyner ağını bozmadan Docker'ın dinamik iptables/nftables manipülasyonlarıyla sorunsuz bir şekilde bir arada çalışır. |
| **Step 1: Prepare Workspace**<br>Create an isolated deployment directory and fetch the production-grade scripts from your official hardening repository.<br><br><code>mkdir -p ~/aegis-firewall-installer && cd ~/aegis-firewall-installer</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/FIREWALL/aegis-firewall-secure.sh</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/FIREWALL/aegis-firewall-lockdown.service</code> | **Adım 1: Çalışma Alanını Hazırlayın**<br>İzole bir dağıtım dizini oluşturun ve resmi katılaştırma deponuzdan üretim seviyesindeki betikleri çekin.<br><br><code>mkdir -p ~/aegis-firewall-installer && cd ~/aegis-firewall-installer</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/FIREWALL/aegis-firewall-secure.sh</code><br><code>curl -O https://raw.githubusercontent.com/Ozhana/Fedora_Hardening/main/FIREWALL/aegis-firewall-lockdown.service</code> |
| **Step 2: Install Core Script**<br>Move the main security script to the system execution path, enforce strict ownership, and make it executable by root only.<br><br><code>sudo cp aegis-firewall-secure.sh /usr/local/bin/aegis-firewall-secure.sh</code><br><code>sudo chown root:root /usr/local/bin/aegis-firewall-secure.sh</code><br><code>sudo chmod 700 /usr/local/bin/aegis-firewall-secure.sh</code> | **Adım 2: Çekirdek Betiği Kurun**<br>Ana güvenlik betiğini sistem yürütme yoluna taşıyın, sıkı sahiplik hakları atayın ve yalnızca root tarafından çalıştırılabilir yapın.<br><br><code>sudo cp aegis-firewall-secure.sh /usr/local/bin/aegis-firewall-secure.sh</code><br><code>sudo chown root:root /usr/local/bin/aegis-firewall-secure.sh</code><br><code>sudo chmod 700 /usr/local/bin/aegis-firewall-secure.sh</code> |
| **Step 3: Deploy Systemd Service**<br>Deploy the lockdown service to hook into the early boot targets, ensuring security rules are active before any network interfaces fully bind.<br><br><code>sudo cp aegis-firewall-lockdown.service /etc/systemd/system/aegis-firewall-lockdown.service</code><br><code>sudo chown root:root /etc/systemd/system/aegis-firewall-lockdown.service</code><br><code>sudo chmod 644 /etc/systemd/system/aegis-firewall-lockdown.service</code> | **Adım 3: Systemd Servisini Dağıtın**<br>Ağ arayüzleri tamamen ayağa kalkmadan önce güvenlik kurallarının aktif olmasını sağlamak için kilitleme (lockdown) servisini erken önyükleme hedeflerine bağlayın.<br><br><code>sudo cp aegis-firewall-lockdown.service /etc/systemd/system/aegis-firewall-lockdown.service</code><br><code>sudo chown root:root /etc/systemd/system/aegis-firewall-lockdown.service</code><br><code>sudo chmod 644 /etc/systemd/system/aegis-firewall-lockdown.service</code> |
| **Step 4: Enable and Initialize**<br>Reload the systemd manager configuration, enable the firewall lockdown service to persist across reboots, and trigger it immediately.<br><br><code>sudo systemctl daemon-reload</code><br><code>sudo systemctl enable --now aegis-firewall-lockdown.service</code> | **Adım 4: Etkinleştirin ve Başlatın**<br>Systemd yönetici yapılandırmasını yeniden yükleyin, güvenlik duvarı kilitleme servisini yeniden başlatmalarda kalıcı olacak şekilde etkinleştirin ve hemen tetikleyin.<br><br><code>sudo systemctl daemon-reload</code><br><code>sudo systemctl enable --now aegis-firewall-lockdown.service</code> |
| **Step 5: Verification & Logs**<br>Verify the structural integrity, active rulesets, and review the boot logs to ensure zero errors during initialization.<br><br><code>sudo systemctl status aegis-firewall-lockdown.service</code><br><code>sudo journalctl -u aegis-firewall-lockdown.service --no-pager -b</code><br><code>sudo nft list ruleset</code> | **Adım 5: Doğrulama ve Loglar**<br>Yapısal bütünlüğü, aktif kural kümelerini doğrulayın ve başlatma sırasında sıfır hata olduğundan emin olmak için önyükleme loglarını inceleyin.<br><br><code>sudo systemctl status aegis-firewall-lockdown.service</code><br><code>sudo journalctl -u aegis-firewall-lockdown.service --no-pager -b</code><br><code>sudo nft list ruleset</code> |
| **⚠️ DOCKER COMPATIBILITY NOTICE**<br>Aegis Firewall V7.2 handles native routing tables beautifully. However, if Docker flushes the system chains during a daemon restart, simply re-apply the shield rules without losing connections:<br><br><code>sudo systemctl restart aegis-firewall-lockdown.service</code> | **⚠️ DOCKER UYUMLULUK NOTU**<br>Aegis Firewall V7.2, yerel yönlendirme tablolarını mükemmel bir şekilde yönetir. Ancak Docker, bir servis yeniden başlatması sırasında sistem zincirlerini temizlerse, bağlantılarınızı kaybetmeden kalkan kurallarını yeniden uygulayın:<br><br><code>sudo systemctl restart aegis-firewall-lockdown.service</code> |
