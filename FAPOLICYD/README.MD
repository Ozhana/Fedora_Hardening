# Fapolicyd Safe Authorizer
## Zero-Trust Application Execution & Telemetry Framework

| ENGLISH  | TÜRKÇE |
| :--- | :--- |
| **1. What is this project?**<br><br>This framework deploys **fapolicyd** (File Access Policy Daemon) on your system. It acts as an ultimate shield, monitoring every application trying to run. Currently, it is set to **Permissive (Observation) Mode** for 14 days. This means it will not block your daily apps (like Docker, Xournal++, or browsers) but will quietly log what *would* be blocked.<br><br>This is a "Zero-Trust" and "Zero-Entropy" deployment. It takes full backups before touching anything and automatically reverts if an error occurs. | **1. Bu proje nedir?**<br><br>Bu altyapı, sisteminize **fapolicyd** (Dosya Erişim Politikası Hizmeti) kurar. Çalışmaya çalışan her uygulamayı denetleyen nihai bir kalkandır. Şu an için 14 günlük **Gözlem (Permissive) Modunda** çalışacak şekilde ayarlanmıştır [cite: 228]. Yani günlük uygulamalarınızı (Docker, Xournal++ vb.) engellemez [cite: 35, 229], ancak neyi engelleyeceğini sessizce kaydeder [cite: 230].<br><br>Bu kurulum "Sıfır Güven" ve "Sıfır Entropi" prensibiyle çalışır [cite: 226]. Sisteme dokunmadan önce tam yedek alır ve hata olursa otomatik olarak geri döner [cite: 170, 193]. |

Paste the Code:** Copy the V8-LTS script code from the source and paste it into the terminal (Right-click -> Paste).<br><br>**D. Run the Script:**<br>`sudo ./aegis-fapolicyd.sh`<br>*(Type your user password if asked. It won't show characters while typing, this is normal).* | **3. | **5. How to Read the Daily Reports**<br><br>The system will now quietly monitor your apps. Every night at exactly 23:59, it will generate a clean report of any apps that triggered the security rules.<br><br>**To view the daily report, type:**<br>`cat /var/log/aegis_telemetry.report`<br><br>**What you need to do:**<br>For the next 7 to 14 days, check this report daily. It will show us which safe apps (like your teacher notes app or docker tools) need to be "trusted" before we fully lock down the system into Enforcing mode. | **5. Günlük Raporlar Nasıl Okunur?**<br><br>Sistem artık uygulamalarınızı sessizce izleyecek [cite: 236]. Her gece tam 23:59'da güvenlik kurallarını tetikleyen uygulamaların temiz bir özetini çıkaracaktır [cite: 233, 239].<br><br>**Günlük raporu görmek için şunu yazın:**<br>`cat /var/log/aegis_telemetry.report`<br><br>**Ne yapmanız gerekiyor?**<br>Önümüzdeki 7 ila 14 gün boyunca bu raporu günlük olarak kontrol edin [cite: 228]. Sistemi tamamen kilitlemeden (Enforcing) önce hangi güvenli uygulamalara (öğretmen not uygulamanız veya docker araçları gibi) "güvenmemiz" gerektiğini bize gösterecektir [cite: 236, 246]. |
| **6. Emergency Rollback (Safety Net)**<br><br>Are you worried about breaking your computer? Don't be.<br>The script has a built-in "Fail-Fast & Rollback" trap. If any step fails during the installation, the script will automatically stop, undo every change it made, and restore your system configuration to the exact moment before you ran it. You don't have to fix anything manually. | **6. Acil Durum Geri Dönüşü (Güvenlik Ağı)**<br><br>Bilgisayarınızı bozmaktan mı korkuyorsunuz? Korkmayın.<br>Betik yerleşik bir "Fail-Fast & Rollback" (Anında Çök ve Geri Al) tuzağına sahiptir [cite: 84]. Kurulum sırasında herhangi bir adım başarısız olursa, betik otomatik olarak durur, yaptığı her değişikliği geri alır ve sistem yapılandırmanızı çalıştırmadan önceki ana tam olarak geri döndürür [cite: 170, 193]. Manuel olarak hiçbir şeyi düzeltmenize gerek yoktur. |
