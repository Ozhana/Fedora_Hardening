# Aegis-Network
## Zero-Trust Firewalld Hardening Framework

| ENGLISH | TÜRKÇE |
| :--- | :--- |
| **Overview**<br>Aegis-Network is a strict, transaction-based Zero-Trust firewall hardening architecture designed for modern Linux systems (specifically tested on Fedora 44 with nftables). It prevents dynamic network interfaces (like USB-C docks or public Wi-Fi) from exposing your system by aggressively locking down `firewalld` and routing all traffic to a deaf `DROP` zone, allowing only essential infrastructure like DHCP/DHCPv6. | **Genel Bakış**<br>Aegis-Network, modern Linux sistemleri (özellikle nftables kullanan Fedora 44) için tasarlanmış, katı ve işlemsel (transactional) bir Sıfır Güven (Zero-Trust) güvenlik duvarı tahkimat mimarisidir. USB-C dock veya ortak Wi-Fi gibi dinamik ağ arayüzlerinin sisteminizi tehlikeye atmasını engeller. `firewalld`'yi tamamen kilitler ve temel DHCP/DHCPv6 altyapısı hariç tüm trafiği sağır bir `DROP` bölgesine hapseder. |
| ⚠️ **WARNING: Aggressive Lockdown**<br>This script is not for average home use. It will instantly block KDE Connect, mDNS, local printer discovery, Chromecast, and Samba shares. It is designed for laptops (like Surface Pro) operating in untrusted, hostile network environments. | ⚠️ **UYARI: Agresif Mühürleme**<br>Bu betik standart ev kullanımı için değildir. KDE Connect, mDNS, yerel yazıcı keşfi, Chromecast ve Samba paylaşımlarını anında engeller. Güvenilmeyen ve düşmanca ağ ortamlarında çalışan dizüstü bilgisayarlar (Surface Pro vb.) için tasarlanmıştır. |
| **Prerequisites**<br>- A system running `firewalld`<br>- Root (sudo) privileges<br>- Basic terminal knowledge | **Gereksinimler**<br>- `firewalld` çalıştıran bir sistem<br>- Root (sudo) yetkileri<br>- Temel terminal bilgisi |
| --- | --- |
| 🛠️ **STEP 1: Create the Core Script**<br>Open your terminal. We need to create the main script file. Run the following command:<br><br>`sudo nano /usr/local/bin/aegis-firewall-secure.sh`<br><br>Paste the **V7 script code** into this file. To save and exit, press `Ctrl+O`, `Enter`, and then `Ctrl+X`. | 🛠️ **ADIM 1: Çekirdek Betiği Oluşturma**<br>Terminalinizi açın. Ana betik dosyasını oluşturmamız gerekiyor. Şu komutu çalıştırın:<br><br>`sudo nano /usr/local/bin/aegis-firewall-secure.sh`<br><br>**V7 betik kodunu** bu dosyanın içine yapıştırın. Kaydedip çıkmak için sırasıyla `Ctrl+O`, `Enter` ve `Ctrl+X` tuşlarına basın. |
| 🔐 **STEP 2: Set Permissions**<br>The system needs permission to execute this file. Run this command to make it executable:<br><br>`sudo chmod +x /usr/local/bin/aegis-firewall-secure.sh` | 🔐 **ADIM 2: Yetkileri Ayarlama**<br>Sistemin bu dosyayı çalıştırabilmesi için yetkiye ihtiyacı var. Çalıştırılabilir yapmak için şu komutu girin:<br><br>`sudo chmod +x /usr/local/bin/aegis-firewall-secure.sh` |
| ⚙️ **STEP 3: Create the Systemd Service**<br>We need to ensure this lockdown happens automatically on boot. Create the service file by running:<br><br>`sudo nano /etc/systemd/system/aegis-firewall-lockdown.service`<br><br>Paste the **Systemd Service (V7) code** into this file. Save and exit (`Ctrl+O`, `Enter`, `Ctrl+X`). | ⚙️ **ADIM 3: Systemd Servisini Oluşturma**<br>Bu mühürlemenin cihaz her açıldığında otomatik yapılmasını sağlamalıyız. Servis dosyasını oluşturmak için şunu çalıştırın:<br><br>`sudo nano /etc/systemd/system/aegis-firewall-lockdown.service`<br><br>**Systemd Servis (V7) kodunu** bu dosyaya yapıştırın. Kaydedin ve çıkın (`Ctrl+O`, `Enter`, `Ctrl+X`). |
| 🚀 **STEP 4: Enable and Start**<br>Now, tell the system to recognize the new service and start it immediately. Run these commands one by one:<br><br>`sudo systemctl daemon-reload`<br>`sudo systemctl enable --now aegis-firewall-lockdown.service` | 🚀 **ADIM 4: Aktifleştirme ve Başlatma**<br>Şimdi sisteme yeni servisi tanımasını ve hemen başlatmasını söyleyeceğiz. Bu komutları sırasıyla çalıştırın:<br><br>`sudo systemctl daemon-reload`<br>`sudo systemctl enable --now aegis-firewall-lockdown.service` |
| 🔍 **STEP 5: Verification**<br>To check if the shield is active and functioning correctly, run:<br><br>`sudo systemctl status aegis-firewall-lockdown.service`<br><br>If it says `Active: active (exited)` and displays a success message, your system is securely locked down. | 🔍 **ADIM 5: Doğrulama**<br>Kalkanın aktif ve düzgün çalıştığını kontrol etmek için şunu çalıştırın:<br><br>`sudo systemctl status aegis-firewall-lockdown.service`<br><br>Eğer ekranda `Active: active (exited)` ve bir başarı mesajı görüyorsanız, sisteminiz güvenli bir şekilde mühürlenmiştir. |
| 🗑️ **How to Uninstall (Rollback)**<br>If you want to return your firewall to its standard state, run the following commands:<br><br>`sudo systemctl disable aegis-firewall-lockdown.service`<br>`sudo rm /etc/systemd/system/aegis-firewall-lockdown.service`<br>`sudo rm /usr/local/bin/aegis-firewall-secure.sh`<br>`sudo systemctl daemon-reload`<br><br>Then, reset firewalld to default via GUI or run `sudo firewall-cmd --set-default-zone=public`. | 🗑️ **Nasıl Kaldırılır (Geri Dönüş)**<br>Güvenlik duvarınızı standart haline döndürmek isterseniz, şu komutları çalıştırın:<br><br>`sudo systemctl disable aegis-firewall-lockdown.service`<br>`sudo rm /etc/systemd/system/aegis-firewall-lockdown.service`<br>`sudo rm /usr/local/bin/aegis-firewall-secure.sh`<br>`sudo systemctl daemon-reload`<br><br>Ardından firewalld'yi arayüzden veya `sudo firewall-cmd --set-default-zone=public` komutuyla varsayılana çevirin. |
